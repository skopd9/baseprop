# üîê Magic Link Custom Email Setup Guide

## Why You Need to Configure Supabase

Unlike organization invitations (which we fully control), **magic link emails are generated by Supabase** because they contain secure authentication tokens. Supabase doesn't expose these tokens to us, so we can't send the emails ourselves.

The **only way** to use your beautiful custom template is to:
1. ‚úÖ Configure Supabase to send emails via Resend (SMTP settings)
2. ‚úÖ Update the email template in Supabase dashboard (one-time paste)

**Good news:** This is a one-time setup! Once configured, all your magic link emails will use your beautiful template.

---

## üöÄ Quick Setup (5 minutes)

### Step 1: Get Resend SMTP Credentials

1. Go to [Resend Dashboard](https://resend.com/api-keys)
2. Copy your API key (starts with `re_`)
3. That's it! Resend SMTP uses your API key as the password

**SMTP Settings:**
```
Host: smtp.resend.com
Port: 465 (SSL) or 587 (TLS)
Username: resend
Password: re_your_api_key_here
```

---

### Step 2: Configure Supabase SMTP

1. Go to [Supabase Dashboard](https://supabase.com/dashboard)
2. Select your project
3. Go to **Project Settings** (‚öôÔ∏è icon) ‚Üí **Auth**
4. Scroll down to **SMTP Settings**
5. Click **Enable Custom SMTP**
6. Fill in the settings:

```
Host: smtp.resend.com
Port Number: 465
Sender email: noreply@yourdomain.com
Sender name: Base Prop
Username: resend
Password: re_your_api_key_here
```

> **Important:** Use a verified domain email in "Sender email". If you haven't verified a domain in Resend, use `onboarding@resend.dev` for testing.

7. Click **Save**

---

## ‚ö†Ô∏è Important Notes

### Required Variables in Template

Make sure these Supabase variables stay in your template:
- `{{ .ConfirmationURL }}` - The magic link URL (REQUIRED)
- `{{ .Email }}` - The user's email address
- `{{ .SiteURL }}` - Your site URL (optional)

### Domain Verification

For production, verify your domain in Resend:
1. Go to [Resend Domains](https://resend.com/domains)
2. Add your domain
3. Add DNS records (SPF, DKIM, DMARC)
4. Use `noreply@yourdomain.com` as sender email

### Testing Email

For testing, you can use:
- `onboarding@resend.dev` (Resend's test email)
- Or your verified domain email

## ü§î Why This Way?

**Why not use Netlify function like we do for invitations?**

Good question! Here's the difference:

### Organization Invitations
- ‚úÖ We create the invitation link
- ‚úÖ We control the token
- ‚úÖ We send via Netlify function + Resend
- ‚úÖ Fully customizable

### Magic Links (Authentication)
- ‚ùå Supabase generates the secure token
- ‚ùå Token never exposed to our code
- ‚ùå Supabase sends the email directly
- ‚úÖ Must use Supabase email system

**This is for security!** Supabase doesn't expose auth tokens to prevent security vulnerabilities.

---

## üì¶ Alternative: Supabase Edge Function (Advanced)

If you really don't want to paste HTML in Supabase dashboard, you could:

1. Create a Supabase Edge Function
2. Use a Database Hook to intercept auth events
3. Send custom emails from the Edge Function

**But this is much more complex** and requires:
- Supabase CLI setup
- Edge function deployment
- Database webhook configuration
- More moving parts to maintain

**Recommendation:** Just use SMTP + template paste. It's the official, supported way! üëç

## üÜò Troubleshooting

### Email not arriving?
- Check spam folder
- Verify Resend API key is correct
- Check Resend dashboard for delivery status
- Verify sender email domain

### Email looks plain/unstyled?
- Make sure you saved the HTML template (not just the subject)
- View email in HTML mode (not plain text)
- Try different email client (Gmail, Outlook)

### Magic link not working?
- Check redirect URL in Supabase Auth settings
- Verify `VITE_APP_URL` environment variable
- Check browser console for errors

### Still having issues?
- Check Supabase logs: Dashboard ‚Üí Logs ‚Üí Auth Logs
- Check Resend logs: [Resend Dashboard](https://resend.com/emails)
- Verify SMTP connection is active in Supabase
